## Project Overview

This project analyses obesity prevalence in Scotland using data from the Scottish Health Survey (2013–2016).

The report investigates whether obesity prevalence differs across:

- survey year
- age
- sex
- socio-economic status
- lifestyle factors such as fruit and vegetable intake

The analysis was carried out using **R and Quarto** to produce a reproducible statistical report including visualisations, exploratory analysis and formal statistical modelling.

---

## Repository Structure

The project is divided into two main parts:

### Explanatory Analysis (EA)

The explanatory analysis explores patterns in the data using descriptive statistics and visualisations.

**EA_part1.qmd**

Explanatory analysis for **Question 1**.  
Investigates how obesity prevalence varies across survey years.

**EAforpart2group.qmd**

Explanatory analysis for **Question 2**.  
Explores relationships between obesity and demographic or lifestyle variables.

**EA3pages.qmd**

Combined explanatory analysis containing **both Question 1 and Question 2**.

**SourcecodeEA**

Supporting scripts used to generate plots and summaries for the explanatory analysis.

**introduction-sourcecode**

Code used to produce figures and summaries included in the introduction section of the report.

---

### Formal Analysis (FA)

The formal analysis applies statistical modelling to formally test the research questions.

**FAQ1.qmd**

Formal statistical analysis for **Question 1**, including modelling and interpretation of results.

**FAQ2.qmd**

Formal statistical analysis for **Question 2**, including model output and interpretation.

**Formal Analysis**

Combined formal analysis containing the statistical modelling for **both questions**.

---

### Report Integration

**Conclusions.qmd**

Final conclusions summarising the results and discussing possible future work.

**Jointsource**

Combined source file used when integrating sections of the report.(trial)

**conclusiondraft1EA**

Draft version of the conclusions section created during development of the report.

## Collaborative GitHub Workflow

GitHub was used to support collaborative development of the project.

The workflow involved:

- creating branches to work on specific sections of the analysis
- committing updates, and merging and deleting branches when completed.
- reviewing contributions from other group members
- merging branches into the main branch once sections were complete

This allowed the group to track changes, collaborate effectively and maintain a well organised repository.
